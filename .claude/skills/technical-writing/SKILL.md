# Skill: Technical Writing (기술 문서 카피라이팅)

## 목적
데이터 분석 결과와 기술적 내용을 설득력 있는 카피로 변환합니다.

## 트리거 조건
- 기술적 데이터를 카피로 변환할 때
- 제품/서비스의 기술적 특장점을 설명할 때
- "기술", "스펙", "기능 설명", "제품 설명" 키워드 포함 시

## 실행 절차

### Step 1: 데이터 수집 및 분류
1. 입력 데이터를 분석한다
2. 핵심 기술 포인트를 추출한다
3. 데이터를 카테고리별로 분류한다:
   - 정량적 데이터 (숫자, 통계, 비율)
   - 정성적 데이터 (리뷰, 피드백, 평가)
   - 기술 스펙 (기능, 성능, 호환성)

### Step 2: 3단계 프로세스 적용

#### Emotion Layer (감정 레이어)
```
입력: 기술 데이터
처리: 독자의 감정을 자극하는 표현으로 변환
출력: 호기심을 유발하는 헤드라인/훅
```
- 기술 수치를 체감할 수 있는 비유로 변환
- "~배 빠른", "~% 절감" 등 임팩트 있는 표현 사용
- 독자의 페인포인트와 연결

#### Logic Layer (논리 레이어)
```
입력: 분류된 데이터
처리: 논리적 근거로 구성
출력: 데이터 기반 본문 카피
```
- 구체적 수치 제시 (반올림 금지, 정확한 숫자 사용)
- 비교 데이터 포함 (전/후, 경쟁사 대비)
- 출처 명시

#### Authority Layer (권위 레이어)
```
입력: 사례/추천 데이터
처리: 신뢰도 높은 증거로 구성
출력: CTA(Call to Action) 포함 마무리 카피
```
- 실제 사용자 후기 인용
- 구체적 성과 수치 포함
- 명확한 행동 유도 문구

### Step 3: 프레임워크별 템플릿

#### PAS 기술 카피 템플릿
```markdown
## [Problem] 헤드라인: 독자의 기술적 고통점
본문: 구체적 문제 상황 묘사 (데이터 포함)

## [Agitate] 서브헤드: 문제 방치 시 결과
본문: 악화되는 시나리오 (통계 포함)

## [Solve] 서브헤드: 솔루션 제시
본문: 제품의 기술적 해결 방식 + 성과 데이터
CTA: 행동 유도 문구
```

#### BAB 기술 카피 템플릿
```markdown
## [Before] 헤드라인: 현재의 기술적 한계
본문: AS-IS 상황 묘사 (벤치마크 데이터)

## [After] 서브헤드: 달성 가능한 미래
본문: TO-BE 비전 (성과 데이터)

## [Bridge] 서브헤드: 도달 경로
본문: 제품의 작동 원리 + 도입 프로세스
CTA: 행동 유도 문구
```

#### FAB 기술 카피 템플릿
```markdown
## [Features] 헤드라인: 핵심 기능
본문: 기술 스펙 목록 (정량적 데이터)

## [Advantages] 서브헤드: 차별적 장점
본문: 경쟁 대비 우위점 (비교 데이터)

## [Benefits] 서브헤드: 독자에게 주는 가치
본문: 실질적 혜택 (ROI, 시간 절감 등)
CTA: 행동 유도 문구
```

## 톤앤매너 가이드
| 항목 | 규칙 |
|------|------|
| 문장 길이 | 1문장 최대 40자 (한글 기준) |
| 전문 용어 | 최초 사용 시 괄호 안에 설명 추가 |
| 숫자 표현 | 천 단위 이상은 쉼표 표기 (예: 1,000) |
| 강조 표현 | **볼드** 는 문단당 최대 2회 |
| CTA | 동사로 시작, 구체적 행동 명시 |

## 품질 기준
- 기술적 정확성: 원본 데이터와 일치 여부 확인
- 가독성: Flesch 점수 기준 "쉬움" 이상
- 설득력: 3단계 프로세스 모든 레이어 포함 여부
- CTA 명확성: 독자가 다음 행동을 즉시 알 수 있을 것
