# Skill: Review Checklist (리뷰 체크리스트)

## 목적
생성된 카피의 품질을 체계적으로 검토하고 개선점을 도출합니다.

## 트리거 조건
- 카피 초안 완성 후 검토 요청 시
- "리뷰", "검토", "체크", "피드백" 키워드 포함 시
- `/review` 커맨드 실행 시

## 리뷰 체크리스트

### 1. 프레임워크 준수 점검

#### PAS 체크리스트
- [ ] **Problem**: 타겟의 핵심 고통점이 명확히 드러나는가?
- [ ] **Problem**: 독자가 "이건 내 얘기다"라고 느끼는가?
- [ ] **Agitate**: 문제의 심각성이 구체적으로 표현되었는가?
- [ ] **Agitate**: 감정적 반응(불안, 긴박함)을 유발하는가?
- [ ] **Solve**: 제품이 문제의 자연스러운 해결책으로 제시되는가?
- [ ] **Solve**: 구체적인 해결 방법이 포함되어 있는가?

#### BAB 체크리스트
- [ ] **Before**: 현재 상황의 불편함이 생생히 묘사되는가?
- [ ] **Before**: 독자가 공감할 수 있는 시나리오인가?
- [ ] **After**: 변화된 미래가 매력적으로 그려지는가?
- [ ] **After**: 구체적이고 측정 가능한 변화를 제시하는가?
- [ ] **Bridge**: Before에서 After로의 경로가 명확한가?
- [ ] **Bridge**: 제품의 역할이 자연스럽게 연결되는가?

#### FAB 체크리스트
- [ ] **Features**: 핵심 기능이 빠짐없이 나열되었는가?
- [ ] **Features**: 기능 설명이 정확하고 구체적인가?
- [ ] **Advantages**: 경쟁 대비 차별점이 명확한가?
- [ ] **Advantages**: "왜 이것인가"에 대한 답이 되는가?
- [ ] **Benefits**: 독자 입장의 실질적 가치가 표현되었는가?
- [ ] **Benefits**: 감정적 + 실용적 혜택이 모두 포함되었는가?

### 2. 3단계 프로세스 점검

#### Emotion 레이어
- [ ] 첫 문장에서 호기심을 유발하는가?
- [ ] 예상치 못한 관점이나 정보가 포함되어 있는가?
- [ ] 독자의 감정적 반응을 이끌어내는가?
- [ ] 스크롤을 멈추게 할 만큼 강력한 훅이 있는가?

#### Logic 레이어
- [ ] 구체적인 데이터/수치가 포함되어 있는가?
- [ ] 논리적 흐름이 자연스러운가?
- [ ] 주장을 뒷받침하는 근거가 충분한가?
- [ ] 출처가 신뢰할 만한가?

#### Authority 레이어
- [ ] 실제 사용자 추천사가 포함되어 있는가?
- [ ] 구체적인 성공 사례가 있는가?
- [ ] 보증/약속이 명확한가?
- [ ] CTA(Call to Action)가 명확하고 강력한가?

### 3. 카피 품질 점검

#### 가독성
- [ ] 문장 길이가 적절한가? (40자 이내 권장)
- [ ] 전문 용어에 설명이 추가되었는가?
- [ ] 단락 구분이 적절한가?
- [ ] 시각적 계층 구조가 명확한가?

#### 톤앤매너
- [ ] 타겟 오디언스에 적합한 어조인가?
- [ ] 일관된 톤을 유지하는가?
- [ ] 브랜드 가이드라인에 부합하는가?

#### 설득력
- [ ] 독자의 행동을 유도하는가?
- [ ] 긴급성(Urgency)이 포함되어 있는가?
- [ ] 희소성(Scarcity)이 활용되었는가?
- [ ] 사회적 증거(Social Proof)가 있는가?

### 4. 기술적 점검
- [ ] 맞춤법/문법 오류가 없는가?
- [ ] 링크가 정상 작동하는가?
- [ ] 이미지/미디어 참조가 올바른가?
- [ ] 메타데이터가 완전한가?

## 리뷰 결과 출력 형식

```json
{
  "review_id": "REV-YYYYMMDD-001",
  "chapter": "챕터명",
  "framework": "PAS|BAB|FAB",
  "overall_score": 85,
  "scores": {
    "framework_compliance": 90,
    "emotion_layer": 80,
    "logic_layer": 85,
    "authority_layer": 80,
    "readability": 90,
    "persuasion": 85
  },
  "passed_items": 18,
  "failed_items": 3,
  "improvements": [
    {
      "section": "영향받는 섹션",
      "issue": "발견된 문제",
      "suggestion": "개선 제안",
      "priority": "high|medium|low"
    }
  ],
  "reviewed_at": "2026-03-18T12:00:00Z"
}
```

## 점수 기준
| 점수 | 등급 | 설명 |
|------|------|------|
| 90-100 | A | 출판 가능, 미세 조정만 필요 |
| 80-89 | B | 우수, 소폭 개선 필요 |
| 70-79 | C | 보통, 주요 섹션 수정 필요 |
| 60-69 | D | 미흡, 상당한 재작성 필요 |
| 0-59 | F | 부족, 전면 재작성 필요 |
